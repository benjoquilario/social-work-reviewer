# 4. Reviewer App Code Walkthrough

## State in ReviewerScreen

- currentIndex: which question is active.
- selectedChoice: user selected option index.
- isSubmitted: prevents changing answer after submit.
- score: number of correct answers.

## Derived values

- currentQuestion = QUESTIONS[currentIndex]
- isFinished = currentIndex >= QUESTIONS.length
- statusText = memoized helper text based on selection/submission state

## Handlers

- handleSubmit
  - Guard: no selection or already submitted -> return.
  - If correct: increment score.
  - Set isSubmitted true.

- handleNext
  - Reset per-question UI state.
  - Move to next question.

- handleRestart
  - Reset all quiz state to initial values.

## Rendering branches

- If finished: show final score + restart button.
- Else: show question card + choices + submit/next flow.

## Why this architecture is good for learning

- Pure local state first (easy to reason about).
- No external state library needed.
- Clear separation between data (QUESTIONS), behavior (handlers), and UI (JSX).
